## KEY USER FLOWS

### Flow 1: New User Looking for Design Workflow
1. Lands on homepage
2. Types "design workflow" in search
3. Sees results with frameworks and tools
4. Clicks "Design Thinking"
5. Reads overview, sees how-to steps
6. Optionally asks AI assistant for specific guidance
7. Saves framework for later

### Flow 2: PM at Growth Stage Needs Metrics
1. Opens app
2. Clicks "Growth" stage card
3. Navigates to "Metrics" tab
4. Browses retention metrics
5. Clicks specific metric for details
6. Sees which guests discussed it
7. Links to related frameworks

### Flow 3: Using AI Assistant
1. Clicks chat icon
2. Types "How do I prioritize features?"
3. AI responds with RICE framework
4. Provides episode references
5. Links to full framework page
6. User can ask follow-up questions

## TECHNICAL CONSIDERATIONS

### Performance
- Lazy load content below fold
- Virtual scrolling for long lists
- Debounced search (300ms)
- Cache API responses
- Optimize images (WebP format)

### Accessibility
- Semantic HTML
- ARIA labels for interactive elements
- Keyboard navigation support
- Screen reader friendly
- High contrast mode support
- Focus indicators visible

### SEO
- Server-side rendering (SSR) for content pages
- Semantic HTML structure
- Meta tags for each page
- Sitemap generation
- Open Graph tags
